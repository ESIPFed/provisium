Implementing W3 PROV AQ Note

This work is an attempt to implement aspects of the W3C Provenance Access and Query Note

https://www.w3.org/TR/prov-aq/

Elements of the Prov AQ Note

  • 01. Accessing

    Directly via a provenance resource location

    Indirectly via query services

  • 02. Locating

    via HTTP with #has_provenance

    via query service found with #has_query_services

    Headers with LINK (note 303s require LINK in redirect target)

    In HTML or RDF (JSON-LD?)

  • 03. Query Service

    Descriptor document (OpenAPI, RDF, schema.org) #ServiceDescriptor

    Direct RPC calls with URI templates and or SPARQL

  • 04. Pingback

    A process to expose much of the other elements in this set while adding in #has_provenance #pingback and #has_query_service
